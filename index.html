Featured Case Study · Computer Vision · Applied ML
NailAI — Fine-Grained Medical Vision System

A fine-grained nail disease classification system designed to balance predictive accuracy, minority-class sensitivity, explainability, and deployable inference.

93%
Accuracy
30%
Minority F1 Lift
5×
Faster Inference
Context

Nail disease classification is difficult because classes are visually similar, long-tail categories matter clinically, and model quality cannot be judged by accuracy alone. The goal was to build a model that performed well while staying practical enough to deploy.

Challenge

The hardest part was balancing three things at once: fine-grained multi-class discrimination, minority-class sensitivity, and inference efficiency suitable for real-world serving.

Approach
  • Compared ResNet18 and ResNet50 under accuracy and latency tradeoffs.
  • Used targeted oversampling and hierarchical classification to improve long-tail behavior.
  • Added Grad-CAM visualization to support explainability and error analysis.
  • Containerized inference for cloud deployment with Docker and Cloud Run.
Why the decisions mattered
  • ResNet18 was selected because it preserved performance while dramatically reducing inference time.
  • Hierarchical structure improved clinically important class distinctions beyond flat classification alone.
  • Explainability was useful both for debugging and for communicating model behavior more responsibly.

Featured Case Study · Robotics · Behavior Cloning
Go1 Autonomous Navigation — Vision-Based Imitation Learning

A vision-based imitation learning system for quadruped navigation, designed to map image observations to control actions while staying measurable through residual diagnostics and trajectory-level evaluation.

BC
Behavior Cloning
ROS
Robot Stack
Vision
Image-Based Policy
Context

This project explored how a quadruped robot could learn navigation behavior directly from visual input. Instead of hand-coding control logic, the goal was to train a policy that predicts motion commands from images.

Challenge

The main challenge was not just training a model, but understanding whether the predicted actions were stable, interpretable, and useful across trajectories. In robotics, small errors can compound quickly, so evaluation needed to go beyond a single aggregate metric.

Approach
  • Used a ResNet-based encoder to process image observations.
  • Trained a behavior cloning policy to predict robot control commands.
  • Evaluated action quality with velocity-wise residual analysis.
  • Analyzed prediction behavior at both sample level and trajectory level.
Why the decisions mattered
  • Image-based encoding kept the perception-to-action pipeline compact and practical.
  • Residual diagnostics made it easier to identify where the policy drifted or underperformed.
  • Trajectory-level analysis gave a more robotics-relevant view than aggregate loss alone.
